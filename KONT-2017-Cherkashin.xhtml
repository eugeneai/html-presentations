Открытые связанные данные (LOD)

  1. Информация доступна в Интернете, причем по лицензии открытых данных;
  2. Представлена в машиночитаемом виде (в Excel а не в виде картинки);
  3. Использован открытый формат (CSV вместо Excel);
  4. Формат базируется на рекомендациях W3C, так, чтобы можно было сослаться на эти данные (RDF, SPARQL);
  5. Из публикуемых данных сделаны ссылки на внешние объекты, чтобы обеспечить контекст.
Приложения получают не просто данные, а отношения между объектами.

Цели и задачи

Средствами LOD + HTML5 в документе формируется логическая разметка, HTML5 + CSS + алгоритмы - различные варианты визуального представления.

Цель - разработка ПО для подготовки и верстки документов, размеченных согласно LOD и порождаемых при помощи LOD,

  • позволяющее создавать документы на рабочей станции пользователя, загружая и преобразуя фрагменты текста из других документов;
  • реализующее различные представления (View): форма ввода и др.
  • Интеграция с другими приложениями обработки LOD.

Использованные онтологии

Friend-of-a-friend (foaf) - информацию об агентах: физических, юридических, программных.
Provenence (prov) - происхождение информации: связь между документами.
Dublin Core (dc) - разметка редактируемой аннотации.
DBPedia resource (dbr) – ссылки на экземпляры объектов и классы.
Schema.org (schema) - объекты, распознаваемые поисковыми агентами Google, Yandex, Yahoo и др. (часть-целое).
The Bibliographic Ontology (bibo) - разметка списка литературы.

Архитектура системы

Реализация

  1. Разметка документа - RDFa, TTL для невидимой части;
  2. JavaScript + библиотека элементов управления;
  3. Компонентная библиотека ZCA для Python (сервер);
  4. Библиотека pymorphy2 + собственная библиотека для склонения имен;
  5. Elasticsearch, Sphinxsearch - системы полнотекстового индексирования;
  6. ClioPatria - Онтологический сервер со встроенной машиной вывода Prolog.

Области применения и дальнейшее развитие

  • Автоматизация подготовки документов;
    • Редактор, учитывающий контекст;
    • Экспорт в офисные форматы;
    • Самоорганизующийся документооборот;
  • Исходные данные для электронных курсов;
    • Импорт в системы автоматизации обучения;
  • Автоматизация разметки
    • Входящей документации (ФОС);
    • Существующих учебный программ.

Заключение

  • Разработан инструментарий для автоматизации верстки документов на основе разметки LOD;
  • Реализованы интерпретаторы логической разметки на стороне клиента (в WEB-браузере);
  • Представлены примеры документов.

Использование LOD в некоторой степени позволяет экономить время, затрачиваемое на проектирование структуры базы данных для хранилищ документов.

Задача актуальна в среде малого бизнеса, где нет устоявшихся информационных потоков и связей между документами, что, в основном, обусловлено динамичностью самого бизнеса.
